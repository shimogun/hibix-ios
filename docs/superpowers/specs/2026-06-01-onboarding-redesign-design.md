# オンボーディング刷新 設計書

- 日付: 2026-06-01
- 対象: `hibix-ios` / `Features/Onboarding`
- ステータス: 承認済み（実装計画へ）

## 1. 目的・コンセプト

オンボーディングを「**機能説明＋取扱説明書を兼ねる**」全9ページのスワイプ式フローに刷新する。アプリの全機能（F-01〜F-14、3つの見守りモード、Pro）を網羅しつつ、終始「**安心を作るアプリ**」として振る舞う。配色は本体と同じ朝焼け水彩テーマ（`Color+HibixTheme`、ライト/ダーク対応）で統一する。

設定画面から**いつでも再閲覧**できるようにし、取説としても機能させる。

## 2. スコープ

### やること
- `OnboardingFlow` を3ページ→**9ページ**に刷新（下記§4）。
- 共通ページレイアウト `OnboardingPageScaffold` を新設し、水彩テーマを適用。
- ⑨開始ページに**見守りモード選択**を実装（Proモード選択時はペイウォール）。
- 設定に「**使い方をもう一度見る**」項目を追加し、再閲覧モードで起動。
- 通知許可ページを廃止（§5参照）。

### やらないこと（据え置き）
- 気分カラー `Color.moodColor`（ブランド §6 確定値）・気分アイコンPNG。
- 既存の課金（StoreKit/Entitlement）・見守りモード・通知スケジューラ・安否メールのロジック本体。オンボーディングからは**既存APIを呼ぶだけ**。
- `daily`（まいにち共有）の内部挙動（v0.1では `gentle` フォールバックのまま）。

## 3. アーキテクチャ・コンポーネント

- `OnboardingFlow`（既存を改修）
  - `enum OnboardingMode { case firstRun, review }` を受け取る。
  - `firstRun`: 全9ページ。⑨で完了処理（`onboarding_done` 書き込み）。
  - `review`: ①〜⑧の情報ページのみ＋「閉じる」。⑨（モード選択・完了）と各種副作用は出さない。
  - `TabView(.page)` でページング。下部に水彩スタイルのページインジケータ＋「次へ」。
- `OnboardingPageScaffold`（新設・共通レイアウト）
  - 構成: 上部イラスト枠 / タイトル（`Color.hibixNavy`, w800）/ サブ（`Color.hibixSubNavy`, w500）/ 任意の追加UIスロット。
  - 背景は `hibixWatercolorBackground()`。
- ページView（新設、`OnboardingPages.swift` を再編 or 分割）
  - `OnboardingConceptPage` … `OnboardingStartPage` の9種。イラストは軽量なSwiftUI描画（SF Symbols＋水彩カラー、ピクセルタイル等）。
- 設定導線
  - `SettingsView` に「使い方をもう一度見る」行を追加 → `OnboardingFlow(mode: .review)` をシート表示。

### 既存連携API（呼ぶだけ）
- 完了: `settingsRepository.setBool(true, forKey: .onboardingDone)`（既存）。
- モード保存: 既存の見守りモード保存経路（`SettingsRepository` / watch_mode）。
- ペイウォール: 既存 `PaywallView` / `PaywallViewModel` をシート表示。
- 通知: Pro購入後セットアップで `notificationScheduler.requestAuthorization()` → `rescheduleDailyNotifications()`（既存メソッド）。

## 5. 挙動・データフロー

### ⑨ 開始ページのモード選択（確定: その場でペイウォール）
- **おひとりさま**: watch_mode=solo を保存 → 完了処理 → ホームへ。
- **ゆるつながり / まいにち共有（Pro限定）**:
  1. ペイウォール（既存 `PaywallView`）をシート表示。
  2. 購入成功 → 当該モードを保存 → 通知許可を要求（許可時のみスケジュール）→ 完了 → ホーム。
  3. キャンセル/未購入 → **おひとりさま**として保存し開始（モードは後から設定で変更可）。

### 通知許可（確定: オンボーディングでは出さない）
- オンボーディングに通知許可専用ページは作らない。
- 許可は「**Pro購入後のセットアップの一部**」として要求（見守り/事前リマインダーの文脈）。
- 無料の朝/夜リマインダーは、設定でONにした時など必要な場面で文脈的に許可要求（本仕様の実装対象外＝既存の設定画面側の責務。オンボーディングからは要求しない）。

### 再閲覧（設定から）
- ①〜⑧を情報ページとして表示。⑧の比較表からはペイウォールへ誘導可（「Proを見る」）。
- ⑨は非表示。完了処理・通知許可・モード保存などの副作用は一切起こさない。「閉じる」で離脱。

## 6. テーマ・アクセシビリティ
- 既存 `Color+HibixTheme` の動的カラー（ライト水彩 / ダーク紺夜空）と共通modifier（背景/カード/ボタン）を流用。外観モードに追従。
- F-10準拠: VoiceOverラベル、Dynamic Type、タップ領域44pt以上を維持/付与。各ページは `accessibilityElement(children: .combine)` を基本とし、操作要素（ボタン/モード選択）は個別にラベル付与。

## 7. エラーハンドリング
- `onboarding_done` 書き込み失敗は `Logger` に記録し、UI上は完了扱いで前進（既存方針踏襲、握り潰さない）。
- ペイウォール購入失敗/キャンセルは「おひとりさまで開始」にフォールバック。
- 通知許可拒否時はスケジュールせず、設定から後で変更可能（既存挙動）。

## 8. テスト方針
- View層は受け入れ基準で担保（手動）。ロジック（モード保存・完了フラグ・分岐）はViewModelに逃がせる範囲でユニットテスト対象に。
- 手動確認: 9ページのスワイプ、ライト/ダーク両表示、⑨各モード（solo即開始 / Proモード→ペイウォール→購入/キャンセル分岐）、設定からの再閲覧（⑨非表示・副作用なし）、VoiceOver読み上げ。
- `xcodebuild build`（iPhone 17 Pro / iOS 26.4）でエラー/警告ゼロ。
